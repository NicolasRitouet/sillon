# Données sensibles

En tant que produit vous allez normalement héberger un minimum de données personnelles. Vous ne pouvez donc pas prendre un hébergeur lambda qui ne répond pas à un certain nombre de critères de confiance. Que ce soit [légalement](legal/data-protection.mdx) ou d'un point de vue [sécurisation des données](/docs/security/).

Pour faire simple, si l'hébergeur affiche les standards suivants vous partez dans une bonne voie :

1.  [SecNumCloud](https://www.ssi.gouv.fr/actualite/lanssi-actualise-le-referentiel-secnumcloud/) ;
2.  [HDS](https://fr.wikipedia.org/wiki/Certification_des_h%C3%A9bergeurs_de_donn%C3%A9es_de_sant%C3%A9) ;
3.  [ISO 27001](https://fr.wikipedia.org/wiki/ISO/CEI_27001) (grand minimum).

À savoir qu'en avril 2023 [l'État annonçait qu'il faut que tout hébergeur se conforme prochainement au SecNumCloud](https://www.economie.gouv.fr/securite-performance-souverainete-strategie-cloud#D%C3%A9velopper%20un%20label%20%C2%AB%20cloud%20de%20confiance%20%C2%BB) pour continuer à héberger des services publics. Cela ne sera pas immédiat puisque certains hébergeurs ont un panel de produits assez large à adapter… Mais si votre hébergeur est HDS c'est déjà un très bon gage de sécurité.

:::note Annuaire
[L'ANSSI référence les hébergeurs SecNumCloud](https://www.ssi.gouv.fr/entreprise/qualifications/prestataires-de-services-de-confiance-qualifies/prestataires-de-service-dinformatique-en-nuage-secnumcloud/), mais cela ne liste pas les petits hébergeurs qui sont basés sur de plus gros (ils ne gèrent pas l'accès physique aux bâtiments…).
:::
